### 2. モジュール別詳細設計

#### ① ネットワーク・タイムモジュール
* **WiFi Manager :** 最初はSSID直書きで設定をさせないために導入します。スマホからWiFi設定できる「WiFiManager」ライブラリを組み込み設定を独立させます。
* **NTP同期:** ESP32では起動時に `configTime()` を行い、JST（日本標準時）を取得します。

#### ② データフェッチモジュール (API部)
* **Open-Meteo API:** 天気の取得ソース。
* **非同期風処理:** 描画を止めないよう、`millis()` を利用したタイマー処理で、15分〜30分おきにデータを更新します。

#### ③ 表示エンジンモジュール (UI部)
* **ステート管理:** 「現在表示中の都市インデックス」を持ち、数秒ごとに切り替える処理をループさせます。
* **色管理:** 天気に応じて背景色を変換して背景色をセットします。

### 3. データ構造 (都市リスト)
日本の主要都市を配列として定義して、管理します。


```cpp
struct CityData {
  const char* name;
  double lat;
  double lon;
};

// 都市リスト（拡張が容易）
CityData cities[] = {
  {"Tokyo", 35.6895, 139.6917},
  {"Osaka", 34.6937, 135.5023},
  {"Okinawa", 26.2124, 127.6761}
};
```

### 4. 画面レイアウト（Phase 1: テキスト版）
128x160ピクセルを以下のように分割します。

* **Header (20px):** 現在時刻 / WiFi強度アイコン
* **Body (100px):** * 中央上：大きなフォントで「気温」（例：25.4℃）
    * 中央下：天気テキスト（例：曇り（Cloudy））
* **Footer (40px):** 表示中の都市名（例：-- 東京 --）
