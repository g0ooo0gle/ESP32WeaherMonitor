# ESP32 Weather Monitor (ST7735 1.8inch)

本プロジェクトは、ESP32とST7735S 1.8インチTFT液晶ディスプレイを使用した、高機能な気象情報ステーションのソースコードです。日本国内の主要都市の気象データを自動巡回表示し、NTPサーバーを利用した正確な時刻同期機能を備えています。

## 1. システム設計仕様

### 1.1 ソフトウェア構成
本システムでは以下のライブラリを採用し、メモリ効率と描画速度の最適化を図っています。

| 機能 | 採用技術 / ライブラリ | 役割 |
| :--- | :--- | :--- |
| 描画エンジン | Adafruit_GFX / ST7735 | 高速なSPI描画制御 |
| 日本語表示 | U8g2_for_Adafruit_GFX | 漢字・ひらがなレンダリング |
| 通信プロトコル | HTTPClient | Open-Meteo APIへのリクエスト |
| データ解析 | ArduinoJson | APIレスポンス(JSON)のパース |
| 時刻管理 | time.h (NTP同期) | configTimeによる日本標準時(JST)の維持 |
| 接続管理 | WiFiManager | キャプティブポータルによるWi-Fi設定の自動化 |

### 1.2 モジュール詳細
* **ネットワーク・タイムモジュール**: WiFiManagerを導入し、SSIDやパスワードのハードコーディングを排除しています。初回起動時にブラウザから設定可能です。
* **データフェッチモジュール**: millis()による非ブロッキングタイマー処理を採用しています。描画処理を中断させず、15分間隔で気象データを更新します。
* **UIエンジン**: 表示都市のステート管理を行い、差分描画によって画面のちらつきを最小限に抑えています。

## 2. 画面レイアウト設計
128x160ピクセルのディスプレイを以下の通り分割し、情報の視認性を向上させています。

* **Header (0-30px)**: NTP同期された正確な時刻（時:分:秒）
* **Body - Top (31-55px)**: 現在表示中の都市名
* **Body - Main (56-121px)**: 気温（小数点第一位まで）
* **Footer (122-160px)**: WMO天気コードに基づく気象状況の説明

---

## 3. ハードウェア配線図
TFTディスプレイとESP32の接続は以下の通りです。

| TFT端子 | ESP32 GPIO | 備考 |
| :--- | :--- | :--- |
| VCC | 3.3V | 電源 |
| GND | GND | グランド |
| CS | 5 | Chip Select |
| RESET | 4 | Hardware Reset |
| D/C (A0) | 2 | Data/Command Selection |
| SDA | 23 | SPI MOSI |
| SCLK | 18 | SPI Clock |
| BL (LED) | 3.3V | バックライト（常時点灯） |

## 4. 今後の開発ロードマップ

### Phase 1: 基本実装 (完了)
* 都市名、気温、天気説明の表示
* Wi-Fi自動接続機能
* 主要10都市の自動巡回表示

### Phase 2: ビジュアル強化(＊一部主要な天気のみ対応)
* WMO天気コードに応じたアイコン表示の実装
* 気象状況に応じた背景色の動的変更

### Phase 3: センサー連携・データ拡張
* 温湿度センサー(BME280等)を用いた屋内環境の表示
* ニュース速報等のテキストティッカー機能の実装

### Phase 4: 高度な機能拡張
* 過去の気温推移グラフの描画
* Webインターフェースによる設定カスタマイズ機能
