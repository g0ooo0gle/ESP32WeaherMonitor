# ESP32 Weather Monitor (ST7735 1.8inch)

本プロジェクトは、ESP32とST7735S 1.8インチTFT液晶ディスプレイを使用した、高機能な気象情報ステーションのソースコードです。日本国内の主要都市の気象データを自動巡回表示し、NTPサーバーを利用した正確な時刻同期機能を備えています。

## 1. システム設計仕様

### 1.1 ソフトウェア構成
本システムでは以下のライブラリを採用し、メモリ効率と描画速度の最適化を図っています。

| 機能 | 採用技術 / ライブラリ | 役割 |
| :--- | :--- | :--- |
| 描画エンジン | Adafruit_GFX / ST7735 | 高速なSPI描画制御 |
| 日本語表示 | U8g2_for_Adafruit_GFX | 漢字・ひらがなレンダリング |
| 通信プロトコル | HTTPClient | Open-Meteo APIへのリクエスト |
| データ解析 | ArduinoJson | APIレスポンス(JSON)のパース |
| 時刻管理 | time.h (NTP同期) | configTimeによる日本標準時(JST)の維持 |
| 接続管理 | WiFiManager | キャプティブポータルによるWi-Fi設定の自動化 |
| RSS解析 | 自前XMLパーサ | NHKニュースRSSのタイトル抽出 |

## 1.2動作フロー
**起動時**: ディスプレイ初期化 → WiFi接続(APモード対応) → NTP同期 → 初回天気取得 → 画面表示 → NEWS背景RSSタスク開始

**ループ**: ボタン検出 → 画面描画 → 裏でデータ更新（15分/1時間/30分ごと）→ NEWS自動ページ送り（5秒ごと）


## 1.3ボタン操作

| ボタン | WEATHER画面 | NEWS画面 |
|---|---|---|
| MODE 短押し | ニュース画面へ切替 | WEATHER画面へ切替 |
| MODE 長押し | 全国巡回 ⇄ 1都市詳細 | 無効 |
| CITY 短押し | 巡回: 都市送り / 1都市: 週間⇄毎時切替 | 次の見出し |
| CITY 長押し | 1都市: 都市送り | 前の見出し |

## 1.4表示モード

```
WEATHER画面
 ├── 全国巡回モード ── 47都道府県を20秒ごとに自動切替
 │                      リージョンフィルタ（全日本/北海道・東北/関東/中部/関西/中国・四国/九州・沖縄）で巡回範囲を制限可能
 └── 1都市詳細モード
      ├── お気に入り都市表示 ── デフォルトは東京、NVSまたはWeb設定で変更可能
      ├── 週間表示 ── 6日間（1時間ごとに更新）
      └── 毎時表示 ── 最大6時間（30分ごとに更新）

NEWS画面 ── NHK RSSから1見出しを全画面表示（5秒で自動切替）
```

## 2. 画面レイアウト設計
128x160ピクセルのディスプレイを以下の通り分割し、情報の視認性を向上させています。

```
┌──────────────────────┐
│ WEATHER画面          │  128×160px
│                      │
│  [時計+都市名]       │  Y= 0〜29
│ ─────────────────    │  Y= 30
│  [現在天気/気温]     │  Y=32〜75
│ ─────────────────    │  Y= 76
│  [週間 or 毎時]      │  Y=78〜159
└──────────────────────┘

┌──────────────────────┐
│ NEWS画面             │
│ ニュース 3/12        │  Y= 0〜19
│ ─────────────────    │  Y= 20
│ 見出し本文 (ページ)  │  Y=22〜159
└──────────────────────┘
```

## 3. ハードウェア配線図
全体構成は以下のとおりです。
| 部品 | 接続 |
|---|---|
| TFT液晶 (ST7735) | CS=GPIO5, RST=GPIO4, DC=GPIO2, SDA=GPIO23, SCLK=GPIO18 |
| MODEボタン | GPIO0 ← → GND |
| CITYボタン | GPIO35 ← → GND |

TFTディスプレイとESP32の接続は以下の通りです。

| TFT端子 | ESP32 GPIO | 備考 |
| :--- | :--- | :--- |
| VCC | 3.3V | 電源 |
| GND | GND | グランド |
| CS | 5 | Chip Select |
| RESET | 4 | Hardware Reset |
| D/C (A0) | 2 | Data/Command Selection |
| SDA | 23 | SPI MOSI |
| SCLK | 18 | SPI Clock |
| BL (LED) | 3.3V | バックライト（常時点灯） |

## 4. 主な機能

### 4.1 気象情報表示
- **Open-Meteo API**から天気コード、気温データを取得
- 天気コードに応じた**アイコン描画**（晴れ/曇り/雨/雪/雷雨etc.）
- 天候に応じた**背景色の動的変更**（5種類）：快晴・曇り・雨・雪・雷雨

### 4.2 リージョンフィルタ
7つのリージョングループに都市を分類し、全国巡回時の表示範囲を制限できます。
- 全日本（47都市）、北海道・東北（7）、関東（7）、中部（9）、関西（7）、中国・四国（9）、九州・沖縄（9）

### 4.3 ニュース表示
- **NHK RSS**（https://www3.nhk.or.jp/rss/news/cat0.xml）からタイトルを取得
- ESP32 Core 0で5分ごとにバックグラウンド取得
- 最大20件の見出しを5秒ごとに自動ページ送り、手動操作でも前後切り替え可能

### 4.4 Web設定サーバー
- ESP32のIPアドレスにアクセス (`http://<ESP32_IP>/`)
- お気に入り都市（1都市詳細モードの初期都市）とリージョンフィルタを設定
- 設定はNVSに永続化されます

### 4.5 描画最適化
- 時計、気温、天気コードの変化がある場合のみ描画を更新（差分更新）
- 不要な画面クリアを削減し、表示のちらつきを低減
