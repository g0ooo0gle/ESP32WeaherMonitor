# ESP32 Weather Monitor

ESP32 と ST7735S 1.8 インチ TFT 液晶ディスプレイを使用した気象情報ステーションです。  
日本全国 47 都道府県庁所在地の天気を表示し、NHK ニュース速報も確認できます。

---

## 1. 機能一覧

| 機能 | 詳細 |
| :--- | :--- |
| **WiFi 自動接続** | WiFiManager でキャプティブポータル設定（初回のみスマホから設定） |
| **NTP 時刻同期** | JST（UTC+9）、秒単位で表示。日付・曜日も表示 |
| **現在天気** | Open-Meteo API から気温・天気コードを 15 分ごとに取得 |
| **週間予報** | 最高・最低気温＋天気アイコンを 6 日分表示（1 時間ごとに更新） |
| **毎時予報** | 気温＋天気アイコンを 6 時間分表示（30 分ごとに更新） |
| **自動画面切替** | 週間 ⇄ 毎時を 1 分ごとに自動切替（両モード共通） |
| **地方巡回モード** | 選択した地方の都市を週間→毎時→次の都市の順に自動巡回 |
| **自分の都市モード** | Web 設定で登録した都市を固定表示。週間↔毎時を自動切替 |
| **NHK ニュース** | RSS から最大 20 件の見出しを取得。タイトル静止表示＋概要スクロール |
| **Web 設定ページ** | ブラウザから都市・地方フィルタを変更して NVS に保存 |

## 2. ハードウェア

### 使用部品

- ESP32 開発ボード（ESP32-WROOM-32 系）
- ST7735S 1.8 インチ TFT 液晶（128×160px）
- 押しボタン 3 個
- 抵抗 10kΩ × 2（GPIO 34・35 用プルアップ）

### ディスプレイ配線（ST7735S）

| TFT 端子 | ESP32 GPIO | 備考 |
| :--- | :--- | :--- |
| VCC | 3.3V | 電源 |
| GND | GND | グランド |
| CS | 5 | Chip Select |
| RESET | 4 | Hardware Reset |
| D/C (A0) | 2 | Data/Command |
| SDA | 23 | SPI MOSI |
| SCLK | 18 | SPI Clock |
| BL | 3.3V | バックライト常時点灯 |

### ボタン配線

| ボタン名 | GPIO | 配線 | 備考 |
| :--- | :--- | :--- | :--- |
| SCREEN | 0 | GPIO 0 ↔ GND | 内部プルアップ使用 |
| MODE | 35 | GPIO 35 ↔ GND | **外付け 10kΩ プルアップ必要** |
| NEXT | 34 | GPIO 34 ↔ GND | **外付け 10kΩ プルアップ必要** |

> **重要**: GPIO 34・35 は入力専用ピンのため内部プルアップが動作しません。  
> 10kΩ 抵抗で 3.3V に接続する外付けプルアップが必ず必要です。

```
外付けプルアップの配線（MODE・NEXT 共通）:

  3.3V ──[10kΩ]──┬── GPIO 34 または 35
                  │
              ボタン一方
                  │
                 GND（ボタン他方）
```

## 4. ボタン操作

3 ボタン・短押しのみのシンプルな操作体系です。

| ボタン | WEATHER（地方巡回） | WEATHER（自分の都市） | NEWS |
| :--- | :--- | :--- | :--- |
| **SCREEN** | ニュース画面へ | ニュース画面へ | 天気画面へ |
| **MODE** | → 自分の都市モードへ | → 地方巡回モードへ | 無効 |
| **NEXT（週間表示中）** | 同じ都市の毎時へ切替 | 毎時へ切替 | 次の見出し |
| **NEXT（毎時表示中）** | 次の都市（週間）へ | 週間へ切替 | 次の見出し |

> NEXT ボタンで手動切替した場合、1 分間は自動切替が発動しません（タイマーがリセットされます）。

## 5. 表示モードと自動切替

```
WEATHER 画面
 ├── 地方巡回モード
 │    週間予報（1分）→ 毎時予報（1分）→ 次の都市 週間予報（1分）→ …
 │    └── NEXT: 週間中→毎時切替 / 毎時中→次の都市へスキップ
 │
 └── 自分の都市モード
      週間予報（1分）→ 毎時予報（1分）→ 週間予報（1分）→ …
      └── NEXT: 週間↔毎時を手動切替（タイマーリセット）

NEWS 画面 ── NHK RSS から最大 20 件を表示
  ├── 上部: 見出し（タイトル）を静止表示（スクロール完了で自動的に次へ）
  └── 下部: 記事概要（ディスクリプション）を電光掲示板スクロール
```

## 6. Web 設定ページ

WiFi 接続後、ブラウザから `http://<ESP32 の IP アドレス>/` にアクセスすると設定画面が開きます。

### 設定項目

| 項目 | 内容 |
| :--- | :--- |
| **自分の都市** | 自分の都市モードで表示する都市 |
| **巡回する地方** | 地方巡回モードで巡回するエリア |

#### 自分の都市の設定方法

| モード | 説明 |
| :--- | :--- |
| **登録都市から選ぶ** | 47 都道府県庁所在地のドロップダウンから選択 |
| **カスタム設定** | 都市名・緯度・経度を手入力（任意の地点を指定可能） |

> **ヒント**: 緯度・経度は Google マップで対象地点を右クリック → 座標をコピーで取得できます。

## 7. データ更新タイミング

| データ | 更新間隔 |
| :--- | :--- |
| 現在天気 | 15 分 |
| 週間天気予報 | 1 時間 |
| 毎時天気予報 | 30 分 |
| 週間 ⇄ 毎時 自動切替 | 1 分（地方巡回・自分の都市モード共通） |
| 都市切替（地方巡回） | 毎時表示から 1 分後に次の都市へ自動切替 |
| 時計表示 | 1 秒ごとに更新（HH:MM:SS + 日付） |
| NHK ニュース RSS | 5 分（Core 0 バックグラウンド取得） |
| ニュース自動ページ送り | スクロール完了後（テキスト長に依存） |

> **NEWS 画面表示中の天気更新**  
> NEWS 表示中は Core 0 を RSS 取得専用にするため、天気の定期取得を停止します。  
> 天気画面に戻ると直ちに最新データを取得します。

## 8. 起動フロー

```text
ディスプレイ初期化
 → 起動ローディング画面（進捗バー付き）
 → WiFiManager（初回は AP モードでスマホから設定）    [1/5]
 → NTP 時刻同期（最大 20 秒待機）                    [2/5]
 → NVS から設定読み込み・Web サーバー起動             [3/5]
 → 初回天気データ取得（同期）                         [4/5]
 → 天気画面表示・バックグラウンドタスク起動           [5/5]
 → ループ開始
```

## 9. マルチコア構成

| Core | 担当処理 |
| :--- | :--- |
| **Core 1**（メイン） | ボタン処理・画面描画・Web サーバー・タイマー管理 |
| **Core 0** | NHK RSS 取得タスク・天気データ非同期取得タスク |

Core 0 ↔ Core 1 間の共有データは Mutex および portMUX（クリティカルセクション）で保護し、  
SPI 描画（TFT）は Core 1 のみで実行してバス競合を防いでいます。

`weatherFetchReady` の読み取りには `takeWeatherFetchReady()` を使用し、  
Core 0 の書き込みと同一の `s_fetchMux` で保護したアトミック操作を行います。

## 12. トラブルシューティング

### WiFi に接続できない

- 初回起動時は "ESP32_Weather_Config" という AP が立ち上がります。スマホから接続して SSID とパスワードを設定してください。
- 設定済みの場合は ESP32 を再起動すると自動再接続します。

### ボタン MODE / NEXT が反応しない、または誤動作する

- GPIO 34・35 は内部プルアップが動作しません。10kΩ 抵抗で 3.3V に接続する外付けプルアップが必要です。
- プルアップが未接続の場合、ピンが浮いてランダムに発火します。
- 2 つのボタンが同時に反応する場合は、GPIO 34 と 35 の配線が逆になっているか、短絡していないか確認してください。

### 天気データが表示されない

- シリアルモニタで HTTP エラーコードを確認してください。
- API サーバー（api.open-meteo.com）への HTTPS 接続が必要です。
- NTP 時刻同期に失敗している場合、毎時予報の現在時刻マッチングが失敗することがあります。

### ニュースが "取得中..." のまま

- NHK RSS サーバー（nhk.or.jp）への HTTPS 接続を確認してください。
- RSS 取得は起動後すぐに開始され、5 分ごとに更新されます。

### 文字が豆腐（□）になる

- フォントには JIS 第 1 水準漢字を含む `japanese3` サブセットを使用しています。
- 人名・地名の異体字など、収録外の文字は表示できません。
